> **RG-P3.S2.T7 (tatrman-server):** the 109-case bulk corpus + 21-case e2e corpus, moved here as a
> runnable. It is **not CI-gating yet** — RG-P6 tiers it. The no-cluster tier (corpus validity +
> the pure `report.py` eval core) runs green with no services:
> `python3 -m venv .venv && .venv/bin/pip install pytest pytest-asyncio && .venv/bin/python -m pytest tests/`.
> The live tiers (`run_eval.py` → `ttr-grounding-mcp`; e2e → the agent chat surface) run against a
> deployed stack. `.venv/`, `.pytest_cache/`, `__pycache__/` are gitignored.

Standalone, reference-clock-pinned eval for the deterministic time/geo/money grounding
feature. Runbook + architecture: [`docs/implementation/grounding.md`](../../docs/implementation/grounding.md).

```
eval/grounding/
├── corpus/
│   ├── grounding-cases.json   # bulk tier (≥100, committed) — regenerated by build_corpus.py
│   ├── e2e-cases.json         # ~20 E2E cases (final-SQL shape)
│   └── supplemental.json      # hand-authored extras merged into the bulk corpus
├── build_corpus.py            # consolidates services/{chrono,geo,money} corpora → grounding-cases.json
├── report.py                  # pure eval + aggregation core (unit-tested, no I/O)
├── run_eval.py                # runner: bulk → grounding-mcp, e2e → golem /v2/chat
├── dashboards/                # Grafana + alert DRAFTS (belong in fabric-infra)
├── rollout/                   # per-instance config DRAFT + release checklist (fabric-infra)
└── tests/                     # corpus validity + report-core unit tests (run with no cluster)
```

## Two tiers

- **bulk** — one call per span straight to `grounding-mcp`, asserted at the
  **GroundingResult** level (`status`, `application`, `source`, `sql_preview` fragments).
  Yields the rules-vs-LLM source rate, outcome distribution, latency percentiles.
- **e2e** — ~20 cases through Golem `/v2/chat`, asserting the grounded **condition
  survives** the cascade into the final plan (grounded flag, `grounding_notes`, SQL shape).

Schema + the model-fixture table: [`SCHEMA.md`](SCHEMA.md).

## Running

```bash
# No cluster — validate corpus + pure eval core (CI-safe):
just eval-grounding-test

# Regenerate the consolidated corpus after editing a service corpus / supplemental.json:
just eval-grounding-build

# Against a deployed df-test stack (needs A11/A12 live):
just eval-grounding
#   → eval/grounding/reports/{metrics.json,report.md}
#   gates: bulk pass-rate ≥ 80% AND LLM-fallback rate ≤ 10%
```

`--models` (a `just` arg) restricts bulk scoring to the fixtures the deployed metadata
model can serve; unservable cases are `skipped:model-unavailable`, not failed.
